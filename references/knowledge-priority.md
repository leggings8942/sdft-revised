# knowledge-priority v0.0.1

## 知識優先順位

1. このSKILL.md（最優先・カノニカルソース）
2. ユーザーから直接提供されたデータファイル（実測値）
3. 複数データが競合する場合：観測値 > 集計導出値 > 古いデータ
4. 不確実性を維持し、強制的な断定をしない

## 4層証拠分類

| ラベル | 意味 | 対象変数（例） |
|--------|------|--------------|
| **Observed** | 実データから直接計測 | 生の時系列データ |
| **Derived** | 観測値から確定的に算出 | S, D, H（時系列から計算） |
| **Assumed** | 分析者が設定した仮定 | U, Θ, 𝓣（推定値） |
| **Hypothesis** | 解釈的・推論的推定 | F（U・Θが Assumed の場合）、定性推定の𝓣 |

## 変数の証拠分類デフォルト

| 変数 | デフォルト分類 | 条件 |
|------|--------------|------|
| S | Derived | 時系列データが存在する場合 |
| D | Derived | 時系列データが存在する場合 |
| H | Derived | 時系列データが存在する場合 |
| F | Assumed | U・Θが外部入力の場合 |
| F | Hypothesis | U・Θが定性推定の場合 |
| 𝓣 | Assumed | v0.0.1では常にAssumed |

## データ欠損時の対応

時系列データが存在しない場合、代理変数を Hypothesis ラベルで推定する。
代理変数を使う場合は必ずその旨をレポートの証拠分類表に明記すること。
